# 숙제 관리 봇 (Homework Tracker Bot)

지후, 윤후의 일일 학습 진행을 Telegram + Google Sheets로 관리하는 봇입니다.

## 동작 방식

1. 부모가 Google Sheets에 주간 학습 계획을 입력
2. 매일 저녁 10시, 각 아이의 Telegram으로 오늘 할일이 **항목별 개별 메시지**로 자동 전송
3. 아이는 인라인 버튼(✅완료 / △부분 / ✗못함)으로 체크
4. 체크 후 [💬 메모 추가] 버튼으로 메모 입력 가능 (책이름, 페이지, 특이사항 등)
5. 결과가 Google Sheets에 자동 기록
6. 매일 저녁 11시 부모에게 요약 전송 (미체크 항목은 "미응답"으로 자동 기록)
7. 매주 일요일 9시 30분 알파 포인트 자동 계산 + 리포트 전송

|------|------|------|

- 태블릿에서 지후/윤후 할일이 모두 전송됨 (같은 Chat ID)
- 와이프 폰과 태블릿은 같은 Telegram 계정 (동시 로그인)

## 세팅 가이드 (순서대로)

### Step 1. Telegram 봇 생성

1. Telegram에서 **@BotFather** 검색하여 대화 시작
2. `/newbot` 입력
3. 봇 이름 입력 (예: `지후윤후 숙제봇`)
4. 봇 username 입력 (예: `jihu_yunhu_homework_bot`)
5. **API Token**이 발급됨 → 메모해둘 것

### Step 2. Chat ID 확인

봇 링크: https://t.me/jihu_yunhu_homework_bot

각 사용자의 Telegram에서:

1. 위 링크 클릭하여 봇 채팅방 진입
2. `/start` 입력
3. 봇이 "등록되지 않은 사용자입니다. 당신의 Chat ID: **XXXXXXX**" 라고 응답
4. 이 숫자를 관리자에게 전달

### Step 3. Google Cloud 서비스 계정 생성

1. [Google Cloud Console](https://console.cloud.google.com/) 접속
2. 새 프로젝트 생성 (이름: `homework-bot`)
3. **API 및 서비스** → **라이브러리** → "Google Sheets API" 검색 → **사용 설정**
4. **API 및 서비스** → **라이브러리** → "Google Drive API" 검색 → **사용 설정**
5. **API 및 서비스** → **사용자 인증 정보** → **사용자 인증 정보 만들기** → **서비스 계정**
6. 서비스 계정 이름 입력 (예: `homework-bot`)
7. 생성된 서비스 계정 클릭 → **키** 탭 → **키 추가** → **새 키 만들기** → **JSON**
8. 다운로드된 JSON 파일을 `credentials.json`으로 이름 변경
9. `homework-bot/` 폴더에 복사

### Step 4. Google Sheets 준비

1. Google Sheets에서 새 스프레드시트 생성
2. 기존 엑셀 파일(`지후 윤후 주간 공부 계획표.xlsx`)을 업로드
   - Google Sheets → **파일** → **가져오기** → **업로드** → 엑셀 파일 선택
   - "스프레드시트 바꾸기" 선택
3. **시트 이름 확인**: `지후`, `윤후` 시트가 있어야 함 (봇이 이 이름으로 읽음)
4. 스프레드시트 URL에서 ID 추출:
   ```
   https://docs.google.com/spreadsheets/d/여기가_SHEETS_ID/edit
   ```
5. **공유 설정**: 스프레드시트 → **공유** → `credentials.json` 안의 `client_email` 값을 추가 (편집자 권한)

### Step 5. 환경 변수 설정

```bash
cp .env.example .env
```

`.env` 파일을 열고 값 입력:

```
TELEGRAM_BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrSTUvwxyz
PARENT_CHAT_ID=123456789
PARENT2_CHAT_ID=987654321
JIHU_CHAT_ID=111222333
YUNHU_CHAT_ID=111222333
GOOGLE_SHEETS_ID=1aBcDeFgHiJkLmNoPqRsTuVwXyZ
GOOGLE_CREDENTIALS_FILE=credentials.json
```

### Step 6. 로컬 실행

```bash
cd homework-bot
pip install -r requirements.txt
python bot.py
```

정상 실행 시 로그:
```
Google Sheets 연결 완료
스케줄 등록: 할일 전송 22:00, 부모 요약 23:00, 알파 계산 일요일 21:30
봇 시작!
```

### Step 7. 동작 테스트

1. 아이 계정 Telegram에서 `/start` → 환영 메시지 확인
2. `/today` → 오늘 할일이 항목별 개별 메시지 + 버튼으로 표시
3. ✅완료 버튼 탭 → 메시지가 "✅ ... — 완료" + [💬 메모 추가] 버튼으로 변경
4. [💬 메모 추가] 탭 → 봇이 "메모를 입력해줘:" → 텍스트 입력 → "📝 메모 저장!"
5. Google Sheets → `지후_결과` 시트에 결과 + 메모 기록 확인
6. 부모 Telegram에서 `/summary` → 요약 확인

## 밀린 숙제 (`/catchup`)

주중에 못한 숙제를 주말에 채워서 할 수 있는 기능입니다.

### 동작 방식
1. 아이가 `/catchup` 입력
2. 이번 주 월~어제 중 **미응답/못함** 항목만 필터링하여 표시
3. 각 항목에 [✅완료] [△부분] 버튼 제공
4. 버튼 탭 → **원래 날짜의 Sheets 데이터가 업데이트** (새 행이 아닌 덮어쓰기)
5. 메모 추가도 가능

### 예시
```
📋 지후의 밀린 숙제 — 3개

[월] 국어 — 비문학 (국풀 나라샤) — 🔇미응답
[✅완료] [△부분]

[화] 사회 — 교과내신 (평가서, 자습서) — ✗못함
[✅완료] [△부분]

[수] 수학 — 학원숙제 — 🔇미응답
[✅완료] [△부분]
```

### 알파 포인트 연동
- 일요일 21:30 알파 계산 **전에** catchup으로 채우면 알파에 반영됨
- Sheets 규칙: "해당 주 분량을 주말 활용해서 완료하면 알파 획득 가능"

## 자동 스케줄

| 시간 | 동작 |
|------|------|
| 매일 22:00 | 지후/윤후에게 오늘 할일 자동 전송 |
| 매일 23:00 | 부모에게 요약 전송 + 미체크 항목 "미응답" 자동 기록 |
| 매주 일요일 21:30 | 주간 알파 포인트 계산 + 부모/아이 모두에게 리포트 전송 |

## 알파 포인트 시스템

### 개요
- Sheets의 마지막 컬럼 "알파체크"에 정의된 규칙에 따라 자동 계산
- **매주 일요일 21:30**에 한 번만 계산되어 확정
- 다음 일요일까지 점수 변동 없음

### 규칙 패턴

| 패턴 | 예시 | 동작 |
|------|------|------|
| 주간 완료 (보상+벌점) | "주3회 모두 O 일때 5분, 아니면 -5분" | 3회 모두 완료 → +5분, 아니면 -5분 |
| 주간 완료 (보상만) | "주2회 모두 O 일때 5분" | 2회 모두 완료 → +5분, 아니면 0분 |
| 입력값 기반 | "주2회 모두 입력값이 있을때 5분, 아니면 -5분" | 메모에 입력 있으면 보상 |
| 횟수 비례 | "책 이름이 입력될때마다 5분씩 추가" | 메모에 책이름 쓸 때마다 +5분 |
| 점수 기반 | "시험 점수가 90 이상일때 5분" | 메모에 점수 입력, 90점 이상 시 +5분 |
| 0점 기반 | "0이 입력될때마다 5분씩 추가, 아니면 -5분" | 메모에 0 입력 시 +5분 |

### 리포트 예시
```
⭐ 지후의 주간 알파 리포트
📅 2026-04-07 ~ 2026-04-13

이번 주: +15분
🏆 누적 점수: 45분

📋 상세:
  ✅ 국어/비문학: +5분
     주3회 달성 → +5분
  ❌ 사회/교과내신: -5분
     주3회 중 2회만 완료 → -5분
  ✅ 책읽기/독서: +10분
     2건 입력 → +10분
```

### 알파 결과 저장
봇이 자동으로 "알파점수" 시트를 생성하여 주차별 기록을 저장합니다.

| 이름 | 주차 | 시작일 | 점수 | 상세 |
|------|------|--------|------|------|
| 지후 | 2026년 15주차 | 2026-04-07 | 15 | (상세 내역) |

## Google Sheets 구조

### 계획 시트 (부모가 관리)

시트명: `지후`, `윤후`

| 과목 | 종류 | 세부 항목 | 교재명 | 횟수 | 요일 | 월~일 | 알파체크 |
|------|------|-----------|--------|------|------|-------|----------|
| 국어 | 자기주도 | 비문학 | 국풀 나라샤 | 주 3회 | 월,목,일 | (수기체크) | 주3회 모두 O 일때 5분... |

- **과목~요일**: 봇이 읽는 핵심 데이터 (헤더 기반 자동 감지, 컬럼 순서 달라도 OK)
- **월~일**: 부모가 수기로 사용하는 체크란 (봇은 사용 안 함)
- **알파체크**: 알파 포인트 규칙 (봇이 일요일에 읽어서 계산)

### 결과 시트 (봇이 자동 생성/관리)

시트명: `지후_결과`, `윤후_결과` (봇 첫 실행 시 자동 생성)

| 날짜 | 과목 | 세부항목 | 교재명 | 결과 | 메모 |
|------|------|----------|--------|------|------|
| 2026-04-08 | 국어 | 비문학 | 국풀 나라샤 | 완료 | 15~18페이지 |
| 2026-04-08 | 책읽기 | 독서 | 책읽기 | 완료 | 해리포터 3권 |

결과 값: `완료`, `부분완료`, `못함`, `미응답`

### 알파점수 시트 (봇이 자동 생성/관리)

시트명: `알파점수` (첫 알파 계산 시 자동 생성)

```
homework-bot/
├── bot.py              ← 메인 봇 (Telegram + 스케줄러 + 명령어 핸들러)

├── requirements.txt    ← Python 패키지
├── .gitignore          ← .env, credentials.json 제외
├── Procfile            ← 서버 배포용
└── README.md           ← 이 파일
```

### 다른 PC에서 실행 시

## 문제 해결

| 증상 | 해결 |
|------|------|
| "등록되지 않은 사용자" | `.env`의 Chat ID 확인 |
| Sheets 연결 실패 | `credentials.json` 경로 확인, Sheets API 활성화 확인 |
| 할일이 안 뜸 | 시트 이름이 정확히 `지후`, `윤후`인지 확인 |
| 요일 매칭 안 됨 | 요일 컬럼에 `월,화,수` 형식으로 입력되어 있는지 확인 |
| 봇 충돌 (409 Conflict) | 봇이 중복 실행 중. 기존 프로세스 종료 후 재시작 |
| 미응답으로 뜸 | 23시 전에 `/summary`를 눌렀을 때는 "미체크"로만 표시. 미응답은 23시 스케줄에서만 기록 |
| 노트북 절전 시 봇 중단 | 전원 설정 → 절전 모드 "안 함", 덮개 닫을 때 "아무 작업도 안 함" |
